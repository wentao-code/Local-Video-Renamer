# 双格式页面快照设计

## 目标

为各数据页面提供独立持久化快照。系统同时写入 MessagePack 与 JSON，读取时优先 MessagePack，MessagePack 缺失或损坏时回退 JSON，并保持旧 JSON 快照可迁移读取。

## 存储结构

所有新快照放在 `user_data/snapshots` 下两个互不混用的根目录：

```text
snapshots/
  messagepack/<页面或实体路径>.msgpack
  json/<页面或实体路径>.json
```

统一快照存储组件负责路径规范化、双写、优先读取、格式校验、临时文件原子替换、删除和旧文件迁移。页面服务不直接操作 JSON 或 MessagePack。

## 一致性与兼容

- 每次成功刷新先生成完整内存载荷，再分别原子写入 MessagePack 和 JSON。
- 读取顺序固定为 MessagePack、新 JSON、旧 JSON、数据库重建。
- MessagePack 读取失败而 JSON 成功时，自动修复 MessagePack 副本。
- JSON 读取失败而 MessagePack 成功不阻塞页面。
- 快照保留版本、过滤规则指纹、来源版本和刷新时间。
- 旧快照只作为迁移来源；新写入不再落在旧混合目录。
- 迁移不得阻塞健康接口启动，大批量详情快照采用按需迁移。

## 页面边界

现有演员库、演员详情、番号库、番号详情、数据中心、名作堂详情和天机阁快照迁移到统一存储。

新增持久化页面快照包括：视频库当前查询页、候选演员、候选番号、沧浪阁、演员天梯榜、番号天梯榜、名作堂列表、勋章堂、女王库列表、女王关键词、女王详情和路径库。带查询条件或实体键的页面以稳定指纹作为文件名，避免不同页面状态互相覆盖。

数据中心继续按总览、七个演员指标、两个番号指标、数据看板及区间详情分别保存。

## 天机阁三级拆分

天机阁一档、二档、三档使用 `video_category/tier_1`、`tier_2`、`tier_3` 三个独立快照。客户端请求携带当前档位，切换档位只读取该档位，手工刷新只重建该档位。

暂存分类、同步分类或过滤规则变更若影响多个档位，可以按实际依赖失效多个档位；普通手工刷新不得重建其他档位。

## 刷新与失效

- 页面刷新只写当前页面或当前实体的快照。
- 页面增删改后只失效相关页面族。
- 补全结束按来源依赖失效受影响页面，已打开窗口再按各自接口刷新。
- 读取快照不触发其他页面计算。

## 依赖与降级

使用官方 `msgpack` Python 包。若运行环境暂未安装该包，系统仍可使用 JSON 启动和运行，并记录 MessagePack 不可用状态；安装依赖后自动恢复 MessagePack 优先读写。

## 测试

测试覆盖双写、读取优先级、损坏回退与自修复、旧 JSON 迁移、原子替换、天机阁三级隔离、页面刷新边界、新增页面快照跨服务实例复用，以及现有完整测试集。

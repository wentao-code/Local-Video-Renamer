# 演员黑名单统一规则设计

## 目标

将名作堂 `handle_mark=2` 与系统现有 `hidden_actors` 统一为演员黑名单。黑名单演员保留历史数据，但不得被新增、同步加入演员库，也不得在演员库、名作堂详情、天梯候选、候选库和统一搜索等界面显示。

## 数据规则

- `masterpiece_actors.handle_mark=2` 的演员自动写入 `hidden_actors`，随后从 `masterpiece_actors` 删除；该删除只影响名作堂演员状态表，不删除演员历史资料。
- 已在 `actors` 表中的黑名单演员不删除历史记录，只在读取层隐藏。
- 所有新增入口必须经过数据库黑名单校验；被拦截时返回明确的 `ValueError`，不创建演员记录。
- 名作堂自动同步遇到黑名单演员时不调用 `add_actor`，记录失败原因并保持其隐藏状态。

## 读取规则

- 演员库基础查询排除 `hidden_actors`。
- 名作堂详情使用统一黑名单过滤，兼容历史上仅存在 `handle_mark=2` 但尚未写入 `hidden_actors` 的记录。
- 天梯演员候选、候选库和统一搜索沿用数据库过滤结果，并补充黑名单条件到直接 SQL 查询。
- 直接打开黑名单演员详情时返回“演员已被加入黑名单”，不展示详情页面内容。

## 实现边界

- 不删除演员、补全、作品和历史关联数据。
- 不改变 `handle_mark=0/1` 的含义。
- 黑名单同步在数据库初始化维护阶段执行，并在名作堂页面点击“刷新”时执行；同步过程保持幂等。

## 验证

- `handle_mark=2` 会写入 `hidden_actors` 并从 `masterpiece_actors` 删除。
- 名作堂页面“刷新”会重新读取并清理 `masterpiece_actors` 状态。
- 黑名单演员无法通过数据库、后台服务或 GUI 新增入口加入演员库。
- 已存在的黑名单演员不会出现在演员库、名作堂、天梯候选、候选库和统一搜索中。
- 非黑名单演员的新增、详情和现有流程保持不变。
